# Virtual Terminal — учебный проект

## Сборка и запуск
```bash
g++ main.cpp -o terminal
./terminal
```

## Поддерживаемые команды
| Команда               | Действие |
|-----------------------|----------|
| `help`                | Показать справку по всем командам |
| `exit`                | Выйти из терминала |
| `ls`                  | Вывести содержимое текущей директории (скрытые не показываются) |
| `cd <путь>`           | Перейти в директорию (`..` — на уровень выше, `/` — от корня) |
| `cat <файл>`          | Показать содержимое текстового файла |
| `touch <имя>`         | Создать файл (тип определяется при первом `edit`) |
| `mkdir <имя>`         | Создать директорию |
| `rm <имя>`            | Скрыть файл/директорию (мягкое удаление) |
| `restore <имя>`       | Восстановить скрытую сущность |
| `mv <старое> <новое>` | Переименовать файл/директорию |
| `edit <файл>`         | Редактировать содержимое (полиморфное поведение) |

## Архитектурные решения
- **Плоская иерархия** — базовый класс `Entry`, наследники: `Directory`, `SingleLineFile`, `MultiLineFile`, `PostponedFile`.
- **Скрытость реализована через точку** в начале имени: `hide()` добавляет `"."`, `restore()` убирает. `isHidden()` проверяет первый символ. При `ls` скрытые сущности не отображаются.
- **`PostponedFile`** — создаётся командой `touch`, а при первом вызове `edit` превращается в `SingleLineFile` или `MultiLineFile` по выбору пользователя.
- **Текущий путь** хранится как `std::vector<Directory*>`, вершина стека — текущая директория.
- **Поиск по имени** — метод `find()` в `Directory`, опционально учитывающий скрытые сущности. При `restore` ищет как точное совпадение, так и совпадение без начальной точки.
- **Полиморфизм** — каждая сущность сама определяет своё поведение через виртуальные методы (`cat()`, `edit()`, `listAll()` и др.). Никаких проверок типов в терминале.
- **Использованы `struct`** (все поля и методы публичны), упрощённый синтаксис без `explicit` и `override`.
- **Все классы в отдельных файлах `.cpp`**, подключённых в `main.cpp`. Заголовочные файлы не используются.
- **Виртуальная файловая система** — команды не затрагивают реальную ОС, работают с внутренним деревом объектов.
